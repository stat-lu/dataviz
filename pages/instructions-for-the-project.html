Instructions for the Project

Overview

The final part of this course is a project that you will undertake. In this project, you will tackle a data set of your choosing using the visualization tools that you’ve encountered during this course. There are two ways to tackle this project, either you

  1. start with a research question and then find a data set that can be used to answer this question, or
  2. start with a data set and then prepare a research question based on this data set.

Data Set

Whichever way you choose, you will need to pick a data set and this data set

  1. should be multivariate, i.e. should contain a relatively large number of variables (aim for at least 6),
  2. should have variables measured on different scales, and
  3. should not be one of the data sets that we’ve already tackled in this course.

You are free to choose any data set you like that fulfill these requirements, but be aware that some data sets out there require a considerable amount of data wrangling before they become amenable for use with ggplot2.

Data Sets in R Packages

The by far simplest approach to pick a data set is to use one that is bundled with an R package. There is a vast number of R packages that contain data sets, but here are a few that may be useful in this regard:

  • carData (miscellaneous)
  • faraway (miscellaneous)
  • GLMsData (miscellaneous, statistical modeling data)
  • UsingR (miscellaneous)
  • HistData (classical data sets from data visualization)
  • agridat (agriculture)
  • archdat (archaeology)
  • Ecdat (econometrics)
  • wooldridge (econometrics)
  • poliscidata (political sciences)

Note that while these data sets are easy to access, many of them still require reformatting such as changing variable types, re-leveling of factors, etc.

In addition, there is a summary of data sets in some R packages here that serves as a nice (but limited) overview.

External Data Sets

Even if there is a wealth of data hidden within packages in R, this collection is still dwarfed by the amount of data that is accessible elsewhere. Finding and wrangling with data outside R often requires a bit more work, however, so if you do take this approach be prepared to get your hands dirty.

Here are some sources for data sets outside R:

  • kaggle
  • UCI machine learning repository
  • Statistics Sweden (SCB)
  • DATA.GOV
  • European Data Portal
  • Sveriges dataportal

Format

The general instructions that applied to the assignments apply here too. But in addition to these, please pay attention to the following as well:

  • The report should contain three sections:

    • an introduction, within which you describe and motivate the research question as well as the data set (including where and how to access it).
    • an analysis, which contains the body of your report, including all (properly formatted) visualizations as well as text describing the those visualizations, and
    • a conclusion, which summarizes the main findings from the report.

    You may use subsections within these sections as you please.

  • The report may be no longer than 6 pages.

  • The report may contain at most 8 figures. Aim for quality rather than quantity. Four well-designed and pertinent visualizations is better than eight half-finished designs.

  • At least some of your visualizations must be multivariate. Eight vanilla scatter plots and histograms will not demonstrate that you’ve grasped the contents of this course.

  • The choice of plots must be reasonably varied. Using the same plot type (e.g. box plots) for all figures, for instance, is not adequate.

  • Adding meta commentary (discussing your choices of visualizations) is typically a good idea (although not mandatory).

Animations and Interactive Graphics

The support for animated and interactive graphics in PDF reports is unfortunately not great. Including interactive visualizations are simply not possible (as far as we know). Animated graphics can be included but only be read by a few select PDF readers (Adobe Acrobat, for instance), so if you want to include animated visualizations, only add them in an appendix to your report.

A possible alternative is to use HTML output instead of PDF output when you knit your R Markdown document and then publish your report on a website such as rpubs. In your submission, you then only need to submit a document with the link to the site.

Alternative Formats

It is generally fine to pursue alternative types of formats for your data visualization projects, such as developing a Shiny application with interactive data visualizations or recording a presentation (with slides of data visualizations), instead of writing a report. If you do, however, note that we may not always be able to help you if you end up struggling and that your fellow students will not be required to peer review your work. Also note that the requirements above still apply when it comes to the variation of and type of visualizations.

If you have an alternative idea for your project format, please be in touch with the course instructor to verify that your plan is okay before you pursue it.
